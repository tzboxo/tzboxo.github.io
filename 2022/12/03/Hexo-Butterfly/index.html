一、前期准备

  • 磨刀不误砍柴工, 先将需要用到的环境配置好，下面流程直接使用
  • win+R 输入 cmd 调取系统终端，在此处输入下面命令

1. Github 创建个人仓库

  1. 创建 Github 账号
  2. 创建仓库，仓库名为：用户名.github.io

详细流程自行百度，相关教程茫茫多

2. Git 安装

地址：https://git-scm.com/download/win
检查：git -v 出现版本号则表示安装成功

3. Git 与 Github 绑定

  • 鼠标右击打开 Git Bash
  • 设置user.name和user.email配置信息：
  • 生成ssh密钥文件：
  • 连续三个回车键，默认不设置密码
  • 然后找到生成的.ssh的文件夹中的id_rsa.pub 密钥，将内容全部复制，一般位置：（C:\Users\Administrator.ssh\id_rsa.pub）
  • 打开GitHub_Settings_keys 页面，新建new SSH Key，标题随意，秘钥粘贴上一步复制的内容
  • 在Git Bash中 检测 GitHub 公钥设置是否成功，输入 ssh git@github.com ：

    如上则说明成功。这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

Node 安装

地址：https://nodejs.org/en/
检查node 安装：命令行输入node -v ，出现版本号则安装成功
检查npm 安装：命令行输入npm -v ，出现版本号则安装成功

3. 提交 Github
